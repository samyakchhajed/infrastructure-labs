# Infrastructure Labs

A collection of hands-on infrastructure and DevOps labs built to develop practical engineering skills through implementation, controlled experiments, failure testing, and evidence-based analysis.

The repository is organized by topic. Each section contains focused labs with its own implementation, documentation, and experimental evidence.

## Labs

| Area | Focus |
|---|---|
| [01 — Observability Labs](01-observability-labs/) | Metrics, instrumentation, dashboards, alerting, and alert-delivery failures |

## Approach

The labs in this repository focus on:

- building and configuring the system being studied;
- running controlled experiments rather than only following setup tutorials;
- deliberately creating failure conditions;
- observing system behaviour through relevant tools and interfaces;
- recording findings based on actual evidence;
- documenting limitations instead of presenting local labs as production systems.

Each lab has its own README with the engineering question, implementation, experiments, findings, and reproduction steps.
